## 一、算法目的

在多个可用渠道（AI 服务提供商）之间，根据管理员配置的**优先级（Priority）**和**权重（Weight）**，智能选择最优渠道转发请求，实现：

1. **流量分配**：按权重比例分配请求，支持灰度和负载均衡
2. **故障降级**：按优先级逐级尝试，高优先级失败后自动切换低优先级
3. **零配置友好**：未配置权重时自动均匀分配，无需手动调参

---

## 二、算法原理

### 2.1 选择策略

```
优先级优先 → 同优先级内加权随机
```

- **Priority（优先级）**：决定候选渠道的范围，值越高越优先
- **Weight（权重）**：在相同优先级的渠道间，决定被选中的概率比例

### 2.2 加权随机核心逻辑

采用**累积权重减法**算法：

```
1. 计算所有候选渠道的总权重 totalWeight
2. 生成 [0, totalWeight) 范围内的随机数 R
3. 遍历渠道，R 减去当前渠道权重
4. 当 R < 0 时，当前渠道即为选中结果
```

**数学证明**：每个渠道被选中的概率 = 该渠道权重 / 总权重

### 2.3 平滑因子机制

为解决**小权重区分度不足**问题，引入动态平滑因子：

| 场景 | 判断条件 | 策略 |
|------|---------|------|
| 所有权重为 0 | `sumWeight == 0` | 每个渠道赋予等效权重 100，实现均匀随机 |
| 平均权重过小 | `sumWeight / n < 10` | 权重放大 100 倍，拉开区分度 |
| 权重差异足够 | `sumWeight / n >= 10` | 不放大，按原始权重计算 |

**效果示例**：

- 权重 0, 0, 0 → 各 33.3%
- 权重 3, 5, 2 → 各 30%, 50%, 20%（放大后）
- 权重 100, 200 → 各 33.3%, 66.7%（原始比例）

## 三、算法实现

### 3.1 两种实现模式

| 模式 | 触发条件 | 数据源 | 特点 |
|------|---------|--------|------|
| **内存缓存模式** | `MemoryCacheEnabled = true` | 内存索引 `group2model2channels` | 带平滑因子，性能极高 |
| **数据库直查模式** | 默认模式 | `abilities` 表 | 基础算法，每渠道保底 +10 权重 |

### 3.2 内存缓存模式伪代码

```
function GetRandomSatisfiedChannel(group, model, retry):
    // 步骤 1: 获取候选渠道列表
    channels = group2model2channels[group][model]
    if channels is empty:
        channels = group2model2channels[group][formatModelName(model)]
    
    // 步骤 2: 按优先级分组
    uniquePriorities = extractUniquePriorities(channels)
    sort(uniquePriorities, descending)
    
    // 步骤 3: 确定本次使用的优先级
    targetPriority = uniquePriorities[min(retry, len-1)]
    targetChannels = filter(channels, priority == targetPriority)
    
    // 步骤 4: 计算权重和平滑因子
    sumWeight = sum(targetChannels.weight)
    if sumWeight == 0:
        smoothingFactor = 1
        smoothingAdjustment = 100
        sumWeight = len(targetChannels) * 100
    else if sumWeight / len(targetChannels) < 10:
        smoothingFactor = 100
        smoothingAdjustment = 0
    else:
        smoothingFactor = 1
        smoothingAdjustment = 0
    
    // 步骤 5: 加权随机选择
    totalWeight = sumWeight * smoothingFactor
    randomWeight = rand.Intn(totalWeight)
    
    for channel in targetChannels:
        randomWeight -= channel.weight * smoothingFactor + smoothingAdjustment
        if randomWeight < 0:
            return channel
    
    return error("channel not found")
```

### 3.3 数据库直查模式伪代码

```
function GetChannel(group, model, retry):
    // 步骤 1: 查询 abilities 表
    abilities = SELECT * FROM abilities
                WHERE group = ? AND model = ? AND enabled = true
                AND priority = (SELECT MAX(priority) FROM abilities WHERE ...)
                ORDER BY weight DESC
    
    // 步骤 2: 加权随机选择
    weightSum = 0
    for ability in abilities:
        weightSum += ability.weight + 10  // 保底权重 10
    
    randomWeight = GetRandomInt(weightSum)
    
    for ability in abilities:
        randomWeight -= ability.weight + 10
        if randomWeight <= 0:
            selectedChannelId = ability.channel_id
            break
    
    // 步骤 3: 查询完整渠道信息
    return SELECT * FROM channels WHERE id = selectedChannelId
```

## 四、算法复杂度

| 指标 | 内存缓存模式 | 数据库直查模式 |
|------|-------------|---------------|
| **时间复杂度** | O(n)，n 为同优先级渠道数 | O(m + k)，m 为 DB 查询开销，k 为结果集大小 |
| **空间复杂度** | O(1)（仅临时变量） | O(k)（存储查询结果） |
| **随机性** | 均匀分布 `rand.Intn()` | 均匀分布 `GetRandomInt()` |
| **并发安全** | `sync.RWMutex` 读锁保护 | 每次独立查询，天然安全 |

## 七、算法特性

### 7.1 正确性保证

1. **概率分布正确**：每个渠道被选中概率严格等于其权重占比
2. **必选其一**：随机数范围 [0, totalWeight)，必然落在某个渠道区间内
3. **空集合安全**：候选渠道为空时返回 nil，不 panic

### 7.2 容错机制

| 异常场景 | 处理方式 |
|---------|---------|
| 渠道权重全为 0 | 等效为均匀随机（各渠道权重 100） |
| 渠道不存在于缓存 | 返回数据库一致性错误，提示修复 |
| 渠道已禁用 | 从缓存索引中移除，不参与选择 |
| 所有渠道失败 | retry 递增，降级到低优先级渠道 |

### 7.3 性能优化

1. **内存索引**：`group → model → []channelId` 三级映射，O(1) 查找
2. **批量同步**：定时从 DB 同步到内存，避免每次查询 DB
3. **读写锁**：`sync.RWMutex` 允许多个请求并发读取

## 八、使用示例

### 8.1 场景：主备渠道 + 负载均衡

```
渠道 A: Priority=10, Weight=80  (主渠道，80% 流量)
渠道 B: Priority=10, Weight=20  (主渠道，20% 流量)
渠道 C: Priority=5,  Weight=100 (备渠道，A/B 失败后使用)
```

**预期行为**：
- 首次请求：80% 概率选 A，20% 概率选 B
- A 或 B 失败后重试：切换到 C

### 8.2 场景：未配置权重（默认均匀）

```
渠道 A: Priority=10, Weight=0
渠道 B: Priority=10, Weight=0
渠道 C: Priority=10, Weight=0
```

**预期行为**：各 33.3% 概率，无需手动调参

## 九、注意事项

1. **权重是相对值**：权重 100:200 与 1:2 效果完全相同
2. **权重 0 不等于禁用**：禁用需设置 `Status != Enabled`
3. **缓存同步延迟**：修改权重后，需等待缓存同步周期（可配置频率）
4. **随机数种子**：Go 1.20+ 自动初始化 seed，无需手动设置
5. **并发安全**：内存缓存模式使用读锁，不影响高并发请求
