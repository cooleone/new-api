# 前端路由与菜单系统

## 1. 路由框架：TanStack Router（文件系统路由）

项目使用 **TanStack Router** 的基于文件的路由系统，通过 Rsbuild 插件自动配置。

**配置入口** — `web/default/rsbuild.config.ts`

关键配置（第 75-83 行）：
```ts
tanstackRouter({
  target: 'react',
  autoCodeSplitting: isProd,  // 开发模式不拆分，生产模式按路由代码分割
}),
```

TanStack Router 插件会扫描 `src/routes/` 目录，自动生成 `src/routeTree.gen.ts`。

**Router 实例创建** — `web/default/src/main.tsx`（第 93-97 行）：
```ts
const router = createRouter({
  routeTree,
  context: { queryClient },
  defaultPreload: 'intent',
  defaultPreloadStaleTime: 0,
})
```

## 2. 路由定义文件结构

### 根路由
- **`src/routes/__root.tsx`** — 根路由布局，包含 `ThemeCustomizationProvider`、`NavigationProgress`、`Toaster`，以及 `beforeLoad` 中的 setup 状态检查和 affiliate 代码保存逻辑。

### 首页（Index Route）
- **`src/routes/index.tsx`** — 路径 `/`，渲染 `Home` 组件（来自 `@/features/home`）。这是一个**公开页面**，使用 `PublicLayout`，显示 Hero、Stats、Features、HowItWorks、CTA 等区块，或者后端配置的自定义首页内容（Markdown 或 iframe URL）。

### 自动生成的路由树
- **`src/routeTree.gen.ts`** — TanStack Router 自动生成的文件（1347 行），定义了所有路由的路径映射和父子关系。

**TanStack Router 命名约定说明：**
- `_authenticated` — 以下划线开头 = **pathless layout route**（不贡献路径段，但提供布局和 `beforeLoad` 守卫）
- `(auth)` 和 `(errors)` — 圆括号包裹 = **pathless group route**（仅用于组织文件，不贡献路径段）
- `$section`、`$chatId`、`$provider`、`$modelId` — 美元符号开头 = **动态路由参数**

## 3. 认证守卫

**`src/routes/_authenticated/route.tsx`**

`_authenticated` 布局路由在 `beforeLoad` 中执行认证检查：
1. 如果本地 `auth.user` 不存在 → 重定向到 `/sign-in`
2. 如果本地有用户信息但会话未验证 → 调用 `getSelf()` API 验证
3. 验证失败 → 清除本地缓存并重定向到 `/sign-in`

认证成功后渲染 `AuthenticatedLayout` 组件。

## 4. 侧边栏/菜单系统

### 4.1 架构总览

菜单系统采用 **Vercel/Cloudflare 风格**的"钻入式"（drill-in）侧边栏模式，分为两层：

- **根导航（Root Nav）** — 默认侧边栏，显示在大多数页面
- **嵌套视图（Nested View）** — 当进入特定工作区（如 `/system-settings/*`）时，侧边栏替换为该工作区的上下文导航，并带有 "Back to Dashboard" 返回按钮

### 4.2 关键文件和职责

| 文件 | 职责 |
|------|------|
| `src/components/layout/components/authenticated-layout.tsx` | 认证布局容器：`SidebarProvider` + `AppHeader` + `AppSidebar` + `SidebarInset`（内容区域） |
| `src/components/layout/components/app-sidebar.tsx` | 侧边栏组件：根据 `useSidebarView()` 返回的数据渲染 `NavGroup` 列表，带过渡动画 |
| `src/hooks/use-sidebar-data.ts` | **根导航数据定义**：返回 4 个导航组的 `SidebarData` |
| `src/hooks/use-sidebar-config.ts` | **侧边栏可见性过滤**：基于管理员配置（`SidebarModulesAdmin`）和用户个人配置（`sidebar_modules`）双层 AND 过滤 |
| `src/hooks/use-sidebar-view.ts` | **视图解析**：判断当前 URL 是否匹配嵌套视图，返回对应的导航组 |
| `src/components/layout/lib/sidebar-view-registry.ts` | **嵌套视图注册表**：目前只注册了 `SYSTEM_SETTINGS_VIEW` |
| `src/components/layout/config/system-settings.config.ts` | System Settings 嵌套视图的配置（路径匹配 + 导航组构建） |
| `src/components/layout/components/nav-group.tsx` | 导航组渲染组件：支持普通链接、可折叠子菜单、折叠状态下的下拉菜单、聊天预设动态列表 |
| `src/components/layout/lib/url-utils.ts` | URL 激活状态判断工具 |
| `src/components/layout/types.ts` | 所有导航相关类型定义 |

### 4.3 根导航菜单定义

**`src/hooks/use-sidebar-data.ts`** 定义了 4 个导航组：

**Chat 组（id: `chat`）：**
- Playground → `/playground`
- Chat（动态聊天预设列表，type: `chat-presets`）

**General 组（id: `general`）：**
- Overview → `/dashboard/overview`
- Dashboard → `/dashboard/models`
- API Keys → `/keys`
- Usage Logs → `/usage-logs/common`
- Task Logs → `/usage-logs/task`

**Personal 组（id: `personal`）：**
- Wallet → `/wallet`
- Profile → `/profile`

**Admin 组（id: `admin`，仅管理员可见）：**
- Channels → `/channels`
- Models → `/models/metadata`
- Users → `/users`
- Redemption Codes → `/redemption-codes`
- Subscription Management → `/subscriptions`
- System Settings → `/system-settings/site`

### 4.4 侧边栏可见性控制

`useSidebarConfig` hook 实现了**双层过滤**机制：

1. **管理员层**（`status.SidebarModulesAdmin`）— 权威配置，管理员禁用某项则所有用户都看不到
2. **用户层**（`auth.user.sidebar_modules`）— 更窄的覆盖层，只能进一步隐藏管理员允许的项；`null` 表示不做限制

URL 到配置键的映射定义在 `use-sidebar-config.ts` 的 `URL_TO_CONFIG_MAP` 中。

### 4.5 Admin 组的角色门控

在 `use-sidebar-view.ts` 中（第 53-55 行），`id === 'admin'` 的导航组仅在用户角色 >= `ROLE.ADMIN` 时显示。

### 4.6 嵌套侧边栏视图（System Settings）

当 URL 匹配 `/^\/system-settings(\/|$)/` 时，侧边栏替换为 System Settings 工作区视图：

- **Back 按钮** → 链接到 `/dashboard/overview`，标签为 "Back to Dashboard"
- **导航组** — "System Administration" 组包含 7 个可折叠子菜单：
  - Site & Branding
  - Authentication
  - Billing & Payment
  - Models & Routing
  - Security & Limits
  - Console Content
  - Operations

每个子菜单的项由对应 feature 目录下的 `section-registry.tsx` 文件定义。

## 5. 首页（Home Page）

**路由文件：** `src/routes/index.tsx`  
**组件文件：** `src/features/home/index.tsx`

路径 `/` 渲染 `Home` 组件，它是**公开页面**（不经过 `_authenticated` 布局），使用 `PublicLayout` 渲染。逻辑如下：

1. 加载中 → 显示 "Loading..."
2. 如果后端配置了自定义首页内容（`useHomePageContent()`）：
   - 如果是 URL → 用 iframe 渲染
   - 如果是 Markdown → 用 `Markdown` 组件渲染
3. 否则 → 显示默认着陆页：`Hero` → `Stats` → `Features` → `HowItWorks` → `CTA` → `Footer`

## 6. 顶部导航

**`src/components/layout/config/top-nav.config.ts`** — `defaultTopNavLinks` 默认为空数组，实际导航链接由后端动态提供。`TopNav` 组件在 `src/components/layout/components/top-nav.tsx` 中实现，支持桌面水平导航和移动端下拉菜单。

## 7. 数据流总结

```
路由定义 (src/routes/**)
    ↓ TanStack Router 插件自动生成
routeTree.gen.ts
    ↓ createRouter()
main.tsx → <RouterProvider>
    ↓ 路由匹配
__root.tsx (根布局)
    ↓ 
    ├── index.tsx → Home (公开页面)
    ├── (auth)/* → 登录/注册等
    ├── (errors)/* → 错误页面
    ├── _authenticated/route.tsx → AuthenticatedLayout
    │       ↓
    │       AppSidebar ← useSidebarView()
    │           ↓                    ↓
    │           ├── 根导航: useSidebarData() → useSidebarConfig() 过滤
    │           └── 嵌套视图: sidebar-view-registry.ts 匹配
    │       ↓
    │       SidebarInset → <AnimatedOutlet> → 子路由组件
    └── 其他公开路由 (about, pricing, rankings...)
```

## 8. 关键设计模式

### 8.1 路由守卫分层
- `__root.tsx` → 全局 setup 检查
- `_authenticated/route.tsx` → 认证守卫
- `system-settings/route.tsx` → 嵌套布局守卫

### 8.2 菜单动态性
- 聊天预设通过 `chat-presets` 类型动态渲染
- Admin 组根据角色动态显示
- 双层过滤（管理员 + 用户）控制可见性

### 8.3 嵌套视图模式
- 通过 `sidebar-view-registry.ts` 注册嵌套视图
- URL 匹配触发侧边栏切换
- 提供上下文相关的导航体验

### 8.4 代码分割
- 生产模式启用 `autoCodeSplitting`
- 按路由自动分割代码
- 优化首屏加载性能
